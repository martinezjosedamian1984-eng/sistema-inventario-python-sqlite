# Proyecto Integrador Final

**Sistema de Gestión de Inventario con Python, SQLite y Rich**

Hola, mi nombre es José Damián Martínez y voy a presentar mi Proyecto Integrador Final de Prácticas Profesionalizantes I.

El proyecto consiste en un sistema de gestión de inventario pensado para un pequeño comercio. La idea principal es resolver un problema común: muchos comercios registran sus productos de forma manual o desorganizada, lo que puede generar errores en el control de stock, dificultad para buscar productos y falta de información para tomar decisiones.

Para resolver este problema desarrollé una aplicación de consola en Python. El sistema permite registrar nuevos productos, visualizar los productos cargados, buscar productos por ID, nombre o categoría, actualizar datos, eliminar productos y generar un reporte de productos con bajo stock.

Para guardar la información utilicé SQLite, una base de datos local y liviana. Esto permite que los datos no se pierdan cuando se cierra el programa. También utilicé la librería Rich para mejorar la visualización en la terminal, mostrando menús, paneles y tablas más claras. Además, utilicé datetime para registrar fecha y hora de carga o reportes.

El proyecto fue organizado siguiendo una metodología ágil simulada. Para eso utilicé Trello, donde armé un tablero con columnas como Backlog, Por hacer, En progreso, En revisión y Finalizado. Allí separé las tareas del proyecto, como definir el problema, crear la base de datos, implementar el CRUD, documentar el proyecto y preparar la presentación.

También utilicé Git y GitHub para versionar y publicar el trabajo. El repositorio contiene el código principal, el README, el informe de ética, el archivo requirements.txt y capturas del sistema funcionando.

En cuanto a la ética, el sistema trabaja con datos de productos y no almacena datos personales. De todas formas, en una implementación real sería importante proteger la base de datos, controlar quién puede acceder y evitar cargar información sensible sin autorización.

Durante el desarrollo aprendí a organizar mejor un proyecto, separar el código en funciones, usar una base de datos SQLite, mejorar la interfaz de terminal con Rich, documentar profesionalmente y simular un flujo de trabajo con herramientas colaborativas.

Como mejora futura, me gustaría convertir este sistema en una aplicación web con Flask, agregar exportación de reportes a CSV, gráficos de stock y filtros más avanzados.

En conclusión, este proyecto integra programación, base de datos, documentación, metodología ágil, ética y comunicación profesional. Fue una simulación cercana a cómo se trabaja en un entorno tecnológico real.
